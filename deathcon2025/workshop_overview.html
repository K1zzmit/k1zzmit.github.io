This workshop follows a progressive learning approach - start with foundations, build understanding, then apply those concepts. The first part of this workshop is instructional and can be found within the video. The second part is hands-on exercises using Splunk and real datasets to practice threat hunting techniques.
1
What is Threat Hunting?
  • Define threat hunting in simple terms - proactive vs reactive security
  • Why organizations need threat hunting
  • The difference between SOC monitoring and active hunting
  • Real-world scenarios where threat hunting makes a difference
Types of Threat Hunting
  • Anomaly-Based Hunting: Looking for deviations from normal behavior
  • Intelligence-Driven Hunting: Using IOCs and threat intel to guide searches
  • Hypothesis-Driven Hunting: Testing assumptions about how attackers might operate
  • When to use each approach (with examples)
  • Demo: Interactive visualization comparing all three types
2
What is a Log?
  • Logs are the breadcrumbs of digital activity
  • Why logs matter for security (visibility is everything)
  • Anatomy of a log entry - timestamps, source, event details
  • Common mistakes: not collecting the right logs or collecting too much noise
Types of Logs
  • Windows Event Logs: Security, System, Application logs
  • Sysmon: Enhanced process and network logging
  • Network Logs: Firewall, proxy, DNS
  • Application Logs: Web servers, databases, custom apps
  • Authentication Logs: Login attempts, privilege escalation
Focus on Windows logs since that's your specialty - bring examples they'll actually encounter
3
What is a SIEM?
  • SIEM = Security Information and Event Management (don't just say the acronym!)
  • The SIEM's job: collect, normalize, correlate, and alert
  • Why SIEMs are critical for threat hunting at scale
  • SIEM limitations - it's only as good as your queries
Types of SIEMs
  • Splunk: Industry standard, powerful search language (SPL)
  • Elastic Stack (ELK): Open source, great for large environments
  • Microsoft Sentinel: Cloud-native, Azure integration
  • Chronicle: Google's SIEM, massive scale
  • QRadar, LogRhythm, others: Brief mention of alternatives
  • Why we're focusing on Splunk for this workshop
4
Threat Hunting in Splunk
  • Introduction to Splunk Search Processing Language (SPL)
  • Basic search syntax and operators (index, sourcetype, search, stats)
  • Building effective hunting queries - start simple, refine iteratively
  • Common pitfalls: being too broad (alert fatigue) or too narrow (missing threats)
Hands-On Exercises
Key Takeaways
  • Start with questions, not queries ("What would an attacker do next?")
  • Iterate and refine - hunting is investigative, not automated
  • Document your findings and build repeatable searches
  • Share knowledge with your team - hunting gets better collectively
